StaticHTML Demo

LAMPython Docker 開發環境

這是一個整合 Apache2、Python 3.13、Django 與 MySQL 的本機開發環境。 本頁面位於 src/projects/StaticHTML/，透過 volume 同步至容器內的 /var/www/StaticHTML。

快速開始

  1. 複製環境設定：cp env .env，並修改 DJANGO_PROJECT_NAME
  2. 啟動所有服務：docker compose up -d --build
  3. 設定 hosts（擇一）：
    • macOS：echo -e "127.0.0.1 local.test\n127.0.0.1 demo.local.test" | sudo tee -a /etc/hosts
    • Windows：編輯 C:\Windows\System32\drivers\etc\hosts，加入 local.test 與 demo.local.test
  4. 開啟瀏覽器造訪下方服務位址

服務位址

Django / Apache
http://local.test
本 Demo 頁面
http://demo.local.test
phpMyAdmin
http://127.0.0.1:8080
MySQL
127.0.0.1:3306

目錄結構與 Volume 映射

本機目錄會即時同步至容器，修改程式碼無需重新 build。

docker-lampython/ ├── docker-compose.yml ├── scripts/ # venv helper（py / pip / venv） ├── .env # DJANGO_PROJECT_NAME 等設定 ├── docker/app/ │ ├── Dockerfile │ ├── requirements.txt │ └── conf.d/ → /etc/apache2/conf.d-host/ └── src/ ├── projects/ → /var/www/ (所有 Django 專案) │ ├── DjangoBlog/ │ ├── StaticHTML/ → /var/www/StaticHTML/ (本頁面) │ └── scripts/ → /var/www/scripts/ (Python demo) ├── data/ → /var/lib/mysql/ ├── sql/ → /root/sql/ └── logs/apache2/ → /var/log/apache2/

Container 一覽

Container 用途 Port
lampython_app Apache2 + mod_wsgi + Python 80, 8000
lamp_mysql MySQL 資料庫 3306
lamp_pma phpMyAdmin 管理介面 8080

常用指令

啟動（背景模式，建議首次加 --build）：

docker compose up -d --build

停止並移除 container：

docker compose down

進入 app container 執行 Django 指令：

docker exec -it lampython_app bash
python manage.py migrate
python manage.py createsuperuser

修改 Apache 設定後重啟：

docker exec -it lampython_app apachectl restart

查看 container IP：

docker inspect -f '{{.Name}} - {{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' $(docker ps -aq)

安裝 Python 套件至 venv（/opt/django_env）：

# 安裝單一套件
docker exec -it lampython_app pip install <package_name>

# 明確指定 venv 的 pip
docker exec -it lampython_app /opt/django_env/bin/pip install <package_name>

# 從本機 requirements.txt 批次安裝
docker cp docker/app/requirements.txt lampython_app:/tmp/requirements.txt
docker exec -it lampython_app pip install -r /tmp/requirements.txt

若要永久保留套件，請將套件寫入 docker/app/requirements.txt， 再執行 docker compose up -d --build 重新 build image。

使用 venv 執行 Python

Container 內 venv 位於 /opt/django_env。 專案根目錄提供 scripts/ helper，可在本機直接透過 venv 執行程式。

執行 Python 程式碼或 .py 檔案：

# 一行指令
./scripts/py -c "print('hello')"

# 執行 demo 腳本（src/projects/scripts/ → /var/www/scripts/）
./scripts/py /var/www/scripts/hello.py

# 在 Django 專案目錄執行 manage.py
./scripts/py manage.py migrate
./scripts/py manage.py createsuperuser

進入已啟用 venv 的互動式 shell：

./scripts/venv

# 或在 venv 中執行單一指令
./scripts/venv python manage.py shell

自訂工作目錄（對應 src/projects/ 下的子目錄）：

LAMPYTHON_WORKDIR=/var/www/scripts ./scripts/py hello.py

等效的手動 docker 指令：

docker exec -it -w /var/www/DjangoBlog lampython_app /opt/django_env/bin/python manage.py migrate
docker exec -it lampython_app /opt/django_env/bin/python -c "print('hello')"

使用 helper 安裝套件：

./scripts/pip install requests
./scripts/pip list

Apache 設定檔（conf.d）

docker/app/conf.d/ 下的 *.conf 會透過 volume 同步至容器，並由 Apache 自動載入。 新增或修改設定檔後，執行 apachectl restart 即可生效，無需重新 build image。

  • local.test.conf — Django 專案（local.test）
  • statichtml.conf — StaticHTML Demo 頁面（demo.local.test）

Django 連線 MySQL 設定

在 Django 的 settings.py 中使用以下設定（容器內 hostname 為 mysql）：

import pymysql
pymysql.install_as_MySQLdb()

DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.mysql",
        "NAME": "django",
        "USER": "root",
        "PASSWORD": "admin",
        "HOST": "mysql",
        "PORT": "3306",
        "OPTIONS": {
            "init_command": "SET sql_mode='STRICT_TRANS_TABLES'"
        }
    }
}

新增專案

  1. 在 src/projects/ 建立新專案資料夾
  2. 在 docker/app/conf.d/ 新增對應的 Apache VirtualHost 設定
  3. 重啟 Apache：docker exec -it lampython_app apachectl restart

所有放在 src/projects/ 的專案都會映射到 /var/www/，可直接在容器內存取。
